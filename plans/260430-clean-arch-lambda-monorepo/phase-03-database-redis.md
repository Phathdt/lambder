# Phase 03 — Database & Redis Layer

## Context Links
- Parent: [plan.md](./plan.md)
- Depends on: [phase-02](./phase-02-shared-packages.md)

## Overview
- **Priority**: P0
- **Status**: Not started
- Concrete persistence: Drizzle schema for `users` + `products`, migration tooling, Redis client for JWT whitelist.

## Key Insights
- `pg` driver is preferred over `postgres.js` for Lambda — better tooling for RDS Proxy IAM auth later.
- Lambda + RDS: **one connection per execution context**. Pool with `max=1`, reuse across warm invocations via module-scope singleton.
- Drizzle migrations are SQL files → can be applied via `drizzle-kit push` (dev) or a one-shot `migrate` Lambda (prod).
- Redis: use `ioredis` with `enableOfflineQueue: false` and `maxRetriesPerRequest: 1` for fast Lambda failover.

## Requirements
- Functional
  - `users` table: `id (uuid)`, `email (unique)`, `password_hash`, `created_at`, `updated_at`
  - `products` table: `id (uuid)`, `owner_id (fk users)`, `name`, `description`, `price (numeric)`, `created_at`, `updated_at`
  - Migrations runnable via `pnpm db:migrate`
  - Redis client supports: `setEx(key, ttl, value)`, `get(key)`, `del(key)`, namespaced keys (`jwt:wl:{userId}:{jti}`)
- Non-functional
  - Pool reused across warm Lambda invocations (`globalThis._pgPool`)
  - All times stored in UTC (`timestamp with time zone`)

## Architecture

### Drizzle schema
```ts
// packages/infra/src/db/schema/users.ts
import { pgTable, uuid, varchar, timestamp } from 'drizzle-orm/pg-core';
export const users = pgTable('users', {
  id: uuid('id').primaryKey().defaultRandom(),
  email: varchar('email', { length: 255 }).notNull().unique(),
  passwordHash: varchar('password_hash', { length: 255 }).notNull(),
  createdAt: timestamp('created_at', { withTimezone: true }).notNull().defaultNow(),
  updatedAt: timestamp('updated_at', { withTimezone: true }).notNull().defaultNow(),
});

// packages/infra/src/db/schema/products.ts
export const products = pgTable('products', {
  id: uuid('id').primaryKey().defaultRandom(),
  ownerId: uuid('owner_id').notNull().references(() => users.id, { onDelete: 'cascade' }),
  name: varchar('name', { length: 200 }).notNull(),
  description: text('description'),
  priceCents: integer('price_cents').notNull(),
  createdAt: timestamp('created_at', { withTimezone: true }).notNull().defaultNow(),
  updatedAt: timestamp('updated_at', { withTimezone: true }).notNull().defaultNow(),
});
```

### Connection pool factory
```ts
// packages/infra/src/db/client.ts
import { Pool } from 'pg';
import { drizzle } from 'drizzle-orm/node-postgres';
import * as schema from './schema/index.js';

declare global { var _pgPool: Pool | undefined; }
export const getPool = () => {
  if (!globalThis._pgPool) {
    globalThis._pgPool = new Pool({
      connectionString: process.env.DATABASE_URL,
      max: 1,
      idleTimeoutMillis: 30_000,
      connectionTimeoutMillis: 5_000,
    });
  }
  return globalThis._pgPool;
};
export const getDb = () => drizzle(getPool(), { schema });
```

### Redis client
```ts
// packages/infra/src/cache/redis-client.ts
import Redis from 'ioredis';
declare global { var _redis: Redis | undefined; }
export const getRedis = () => {
  if (!globalThis._redis) {
    globalThis._redis = new Redis(process.env.REDIS_URL!, {
      enableOfflineQueue: false,
      maxRetriesPerRequest: 1,
      lazyConnect: true,
    });
  }
  return globalThis._redis;
};
```

### TokenStore adapter (Redis whitelist)
```ts
// packages/infra/src/cache/redis-token-store.ts
export class RedisTokenStore implements TokenStore {
  constructor(private redis: Redis) {}
  whitelist(userId: string, jti: string, ttlSeconds: number) {
    return this.redis.setex(`jwt:wl:${userId}:${jti}`, ttlSeconds, '1');
  }
  isWhitelisted(userId: string, jti: string) {
    return this.redis.get(`jwt:wl:${userId}:${jti}`).then(v => v === '1');
  }
  revoke(userId: string, jti: string) {
    return this.redis.del(`jwt:wl:${userId}:${jti}`);
  }
  revokeAll(userId: string) {
    // SCAN + DEL pattern; documented as O(N) — acceptable for logout-everywhere
  }
}
```

## Security Considerations
- `password_hash` column never logged. Repositories return entities WITHOUT the hash by default; an explicit `findByEmailWithHash` is used only by the login use case.
- Redis URL from env — never committed.
